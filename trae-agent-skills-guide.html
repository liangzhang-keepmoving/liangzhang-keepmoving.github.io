How to Use Agent Skills in Trae

According to the latest information, Trae supports integrating Claude Skills through the OpenSkills framework for functionality expansion. Here's a complete usage guide:

1. Environment Preparation

First, you need to install the OpenSkills command-line tool, which is the core component for managing Skills:

npm i -g openskills

2. Installing Skills (Three Methods)

Method 1: Install from Anthropic Official Market
# Project-level installation (recommended, stored in ./.claude/skills at project root)
openskills install anthropics/skills

# Global installation (stored in ~/.claude/skills in user directory)
openskills install anthropics/skills --global
Method 2: Install from GitHub Repository

Suitable for third-party or custom Skills:

openskills install your-org/custom-skills
Method 3: Install from Local Path

For private Skills under development:

openskills install /path/to/my-skill

3. Sync Skills to Trae

This is a crucial step! After installation, you must sync to make Trae recognize these Skills:

# Sync to Trae's default rules file AGENTS.md
openskills sync

# Or sync to a custom path (will be created automatically if it doesn't exist)
openskills sync --output .ruler/AGENTS.md

4. Configure Rules File in Trae

Trae loads Skills through rules files. Here's how to do it:

  1. Open Trae's rules configuration: Find "Rules" settings in Trae
  2. Create AGENTS.md file: Create a new `AGENTS.md` file in the rules directory (if you didn't specify a custom path during sync)

5. Using Skills

After configuration, you can directly enter natural language commands in Trae's Agent chat:

Example 1: Using PDF Skill

"Convert the current project's README.md to PDF and place it in the project"

Example 2: Using Excel Skill

"Analyze the data.csv file in the project and generate an Excel report with charts"

6. Advanced: Developing Custom Skills

If existing Skills don't meet your needs, you can develop custom Skills:

my-skill/
├── skill.md          # Skill definition file
├── references/       # Reference documentation
├── scripts/          # Execution scripts
└── assets/           # Resource files

7. Common Management Commands

# List installed Skills
openskills list

# Read details of a specific Skill
openskills read pdf

# Interactively remove Skills
openskills manage

# Remove a specific Skill
openskills remove <skill-name>

Notes

  1. Path Priority: OpenSkills uses a layered loading strategy with priority from high to low:
    • ./.agent/skills/ (project-level - general)
    • ~/.agent/skills/ (global - general)
    • ./.claude/skills/ (project-level - Claude)
    • ~/.claude/skills/ (global - Claude)
  2. Sync Failure Troubleshooting: If Trae can't recognize Skills, check:
    • Whether you correctly executed `openskills sync`
    • Whether `AGENTS.md` is in Trae's rules loading path
    • Whether Skills are installed in the correct directory
