SGLang (Part 3): Ascend NPU Support Analysis

This article analyzes the Ascend NPU support in SGLang, including the backend architecture, key optimization technologies, recent improvements, and performance benchmarking.

Ascend Backend Architecture

SGLang provides complete backend support for Ascend NPU through a dedicated hardware adaptation layer, fully leveraging the performance advantages of Ascend NPU. The Ascend backend architecture mainly consists of the following parts:

1. Hardware Abstraction Layer

The hardware abstraction layer is responsible for direct interaction with Ascend NPU hardware, providing a unified interface:

  • NPU device management: Initialize and manage NPU devices
  • Memory allocation and management: Efficiently allocate and manage NPU memory
  • Stream and event processing: Manage NPU computation streams and events
2. Optimization Components

Optimization components include dedicated optimizations for Ascend NPU features:

  • MLA preprocessing: Matrix lookup acceleration, optimizing attention computation
  • nz format conversion: Convert regular matrix format to NPU-optimized nz format
  • Operator fusion: Fusion multiple operators to reduce kernel launch overhead
  • Memory layout optimization: Optimize data layout in NPU memory
3. Model Adaptation

The model adaptation layer is responsible for adapting general models to run on Ascend NPU:

  • Weight format conversion: Convert model weights to NPU-optimized format
  • Operator mapping and replacement: Map general operators to NPU-specific operators
  • Precision control and optimization: Optimize computation precision according to NPU features

Key Optimization Technologies

            self.q_b_proj_deq_scale,
            self.kv_a_layernorm.weight,
            cos,
            sin,
            self.w_kc,
            k_cache,
            v_cache,
            slot_mapping,

            q_rope_out,
            v_cache,
            q_nope_out,

            forward_batch,
            zero_allocator,

nz Format Conversion

Principle: nz format is an optimized matrix storage format for Ascend NPU, improving memory access efficiency and computation performance by adjusting data layout.

Implementation:

  • Matrix blocking: Divide matrix into fixed-size blocks
  • Data rearrangement: Rearrange data in NPU-friendly order
  • Padding and alignment: Ensure data meets NPU access requirements
def transdata(nd_mat, block_size: tuple = (16, 16)):
    """Convert matrix to nz format

    Args:
        nd_mat: Input matrix
        block_size: Block size, default is (16, 16)

    Returns:
        nz format matrix
    """
    # Calculate padded size
    r = round_up(nd_mat.shape[0], block_size[0])
    c = round_up(nd_mat.shape[1], block_size[1])
    r_pad = r - nd_mat.shape[0]
    c_pad = c - nd_mat.shape[1]

    # Pad matrix
    nd_mat = F.pad(nd_mat, ((0, r_pad, 0, c_pad)))

    # Rearrange data to nz format
    nz_mat = torch.permute(
        torch.reshape(
            nd_mat,
            (r // block_size[0], block_size[0], c // block_size[1], block_size[1]),
        ),
        [2, 0, 1, 3],
    )

    # Reshape to final format
    nz_mat = torch.reshape(
        nz_mat, (nz_mat.shape[0], nz_mat.shape[1] * nz_mat.shape[2], nz_mat.shape[3])
    )

    return nz_mat
Operator Fusion and Optimization

Principle: Operator fusion is the process of merging multiple independent operators into a composite operator, reducing kernel launch overhead and improving computation efficiency.

Implementation:

  • Attention computation fusion: Fusion QKV projection, attention computation and output projection
  • Layer normalization fusion: Fusion layer normalization and linear transformation
  • Activation function fusion: Fusion linear transformation and activation function

Advantages:

  • Reduce kernel launch overhead
  • Improve memory access locality
  • Fully utilize NPU computation resources

Summary

SGLang provides comprehensive backend support for Ascend NPU through a dedicated hardware adaptation layer and optimization components, fully leveraging the performance advantages of Ascend NPU. Core optimization technologies include MLA preprocessing, nz format conversion, and operator fusion, which collectively significantly improve model inference performance on Ascend NPU.

Recent improvements include CI fixes, weight conversion optimization, redundant format conversion removal, and diffusion model support, further enhancing the stability and performance of the Ascend backend. Through reasonable performance tuning and deployment strategies, optimal model inference performance can be achieved on Ascend NPU.

SGLang's Ascend backend implementation provides Ascend NPU users with an efficient, stable LLM inference solution, making Ascend NPU one of the ideal choices for LLM deployment.
