Complete Guide to npm Installation

npm (Node Package Manager) is typically installed with Node.js. Here are the installation methods for different operating systems:

1. Recommended Method: Node.js Official Installer (Simplest)

Windows / macOS
  1. Visit nodejs.org
  2. Download the LTS version (Long Term Support, recommended) or Current version
  3. Run the installer with all default settings
  4. Verify installation: 
    node -v   # Check Node.js version
npm -v    # Check npm version
Linux (Ubuntu/Debian)
# Using NodeSource official repository (recommended)
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# Verify
node -v && npm -v

2. Developer Recommendation: Using Version Management Tools

Option A: nvm (Node Version Manager)

Suitable for developers who need to switch between multiple Node.js versions

# Install nvm:
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# After installation, restart terminal or run
source ~/.bashrc  # or source ~/.zshrc

# Using nvm to install Node.js/npm:
nvm install --lts      # Install latest LTS version
nvm use --lts          # Use LTS version
nvm alias default lts  # Set default version

# Check installed versions
nvm ls

3. Update npm to the Latest Version

npm is installed with Node.js, but it's usually not the latest version. It's recommended to update manually:

# Globally update npm (using npm self-update)
npm install -g npm@latest

# Verify
npm -v

4. Permission Issue Solutions

EACCES errors on Linux/macOS

Recommended Method: Use nvm

nvm automatically handles permission issues without additional configuration

5. Quick Installation Verification

Create a test project to confirm npm is working properly:

# 1. Create project folder
mkdir test-npm && cd test-npm

# 2. Initialize package.json
npm init -y

# 3. Install a test package
npm install lodash

# 4. Check if successful
ls node_modules  # Should see lodash folder

6. Installation Method Comparison

Method Advantages Disadvantages Suitable For
Official Installer Simple and quick, one-click completion Not easy to switch versions Beginners, general users
nvm/fnm Easy multi-version management, permission-friendly Requires additional installation Developers, those needing to test multiple versions
System Package Manager Integrated with system Versions are usually older Linux servers

Recommendation: Developers use nvm, general users directly use the official installer.
